## 🔧 继续翻译其他页面

### 方法：替换硬编码文本

**示例1：简单文本**
```vue
<!-- 之前 -->
<view>确认</view>

<!-- 之后 -->
<view>{{ $t('common.confirm') }}</view>
```

**示例2：动态属性**
```vue
<!-- 之前 -->
<u-navbar title="商品详情"></u-navbar>

<!-- 之后 -->
<u-navbar :title="$t('goods.detail')"></u-navbar>
```

**示例3：JavaScript中**
```javascript
// 之前
uni.showToast({ title: '操作成功' })

// 之后
uni.showToast({ title: this.$t('message.operationSuccess') })
```

## 📝 添加新的翻译

### 步骤1：在语言包中添加

**lang/zh-CN.js**:
```javascript
export default {
  // 添加新分类
  myModule: {
    myKey: '我的文本'
  }
}
```

**lang/en-US.js**:
```javascript
export default {
  // Add new category
  myModule: {
    myKey: 'My Text'
  }
}
```

### 步骤2：在页面中使用

```vue
<view>{{ $t('myModule.myKey') }}</view>
```

## 🐛 常见问题

### Q1: 切换语言后没反应？

**检查**：
1. main.js是否引入i18n
2. Vue实例是否注入i18n
3. 页面是否使用$t()

**解决**：
```bash
# 重新编译
HBuilderX → 停止 → 重新运行
```

---

### Q2: 显示"undefined"或key名？

**原因**：
- 语言包中没有这个key
- key拼写错误

**解决**：
```javascript
// 检查语言包
console.log(this.$t('common.confirm')) // 应该显示"确认"
```

---

### Q3: 新增语言包后没生效？

**解决**：
1. 检查lang/index.js是否import
2. 检查messages中是否添加
3. 重新编译项目
